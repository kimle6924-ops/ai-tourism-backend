## Quy tắc làm việc

- **Trước khi bắt đầu task, ĐỌC các file .md liên quan** (xem mục "Quy tắc đọc tài liệu" bên dưới).
- Chỉ viết code. Người dùng sẽ tự build và test.
- Luôn tuân thủ kiến trúc trong project-overview.md.
- Chỉ sửa các file liên quan trực tiếp đến task.
- Nếu task chưa rõ → hỏi lại trước khi viết code.
- Sau khi code xong → cập nhật documentation liên quan.

## Quy tắc cập nhật documentation

Cập nhật document khi code ảnh hưởng tới:

- database schema → database-design.md
- API endpoint → api-endpoints.md
- trạng thái feature → implementation-status.md

## Quy tắc đọc tài liệu

Chỉ đọc file cần thiết cho task.

Ví dụ:

Task API:
- project-overview.md
- api-endpoints.md

Task database:
- database-design.md

Task feature:
- project-overview.md
- implementation-status.md
